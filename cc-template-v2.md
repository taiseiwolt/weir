# CC依頼テンプレート v2

> Claude Code内部設計パターンを統合した改良版。全CC依頼文はこのテンプレートに従うこと。
> 適用開始: 2026-04-01
> 出典: Claude Code ソース分析レポート S-01〜S-07, S-16

---

## テンプレート本体

```
## 依頼内容
(具体的な作業内容)

### 指示の書き方ルール（S-01: 理解委譲禁止）
- ❌ 禁止: 「調査結果に基づいて修正して」「適切に対応して」「良い感じにして」
- ✅ 必須: ファイルパス・関数名・行番号・条件を明記
- 例: 「weir-order-store.html の goToCheckout() 関数（約L450）にカート空チェックを追加。getCartCount()===0 の場合はアラート表示して return」

### スコープ制限（S-05: やりすぎ防止）
- 依頼範囲外の改善・リファクタリング禁止
- 起こり得ないシナリオのエラーハンドリングを追加しない
- 1回しか使わない処理をヘルパー関数化しない
- 触っていないコードにコメント・型アノテーションを追加しない
- 「似た3行のコードは、早すぎる抽象化より良い」

### コメントルール（S-06: コメント規律）
- デフォルトはコメント不要
- WHYが自明でない場合のみ追加（隠れた制約、特定バグの回避策）
- WHATの説明コメント禁止（良い命名で代替）

## 実行前チェック（省略厳禁）
- [ ] information_schema.columnsで対象テーブルの全カラム確認
- [ ] CHECK制約の有効値確認（pg_constraint）
- [ ] NOT NULLカラムの洗い出し
- [ ] 既存データとの競合確認（SELECT COUNT）
- [ ] 変更対象ファイルが他のCC依頼と競合しないか確認
- [ ] 参照しているファイルパス・関数名が現在も存在するか確認（S-07: メモリを信用しない）

## 品質レベル（UI/フロントエンド変更を含む場合のみ）
- Draft: データが入っていればOK。絵文字・仮テキスト許容
- Review: 全画像・テキスト・リンクが本番品質
- Production: 元サイトとの比較含めて完全チェック

## 完了基準
(具体的な動作・データ・エラーケース)

### UIUXチェック（UI/フロントエンド変更を含む場合は必須）
- [ ] 全クリック可能要素をクリックして「何も起きない」「404」がないこと
- [ ] 仮テキスト（サンプル/テスト/Lorem ipsum/TODO/TBD）が残っていないこと
- [ ] 他ブランド名がハードコードされていないこと
- [ ] 画像が表示されない・壊れている箇所がないこと
- [ ] 既存ブランド（デフォルト）の表示に影響がないこと

## 完了報告ルール

### 事実報告の義務（S-02: 虚偽報告防止）
- テストが失敗したなら「失敗した」と報告せよ。出力を添付せよ
- 検証ステップを実行していないなら「実行していない」と明記せよ。成功を暗示するな
- 出力が失敗を示しているのに「全テストPASS」と主張するな

### 逆方向の禁止（S-03: 過度なヘッジ禁止）
- 確認済みの結果に不要な免責条項を付けるな
- 完了した作業を「部分的」に格下げするな
- 既に確認したことを再検証するな
- 目標は「正確な報告」であり「防衛的な報告」ではない

### 検証の義務（S-04: 完了前検証）
- タスク完了を報告する前に、実際に動くことを検証せよ
- テストを実行し、スクリプトを実行し、出力を確認せよ
- 検証できない場合は「未検証」と明記せよ。成功を主張するな

### 報告フォーマット
- 変更箇所のdiffのみ
- 本番URLでの動作確認結果
- **検証方法の明記（S-16）**: 以下のいずれかを必ず記載
  - 「コードレビュー」= ソースコードを読んで判定
  - 「実操作テスト」= 本番URLでブラウザ操作して確認
  - 「自動テスト」= Playwright等で自動実行
  - 「未検証」= 検証を実行していない
- 手動作業がある場合はリスト
- **Firecrawlスキャン依頼（HTMLファイル変更を含む場合のみ必須）**:
  CCはJavaScript実行後のレンダリング内容を確認できない。
  HTMLファイルを変更した場合、完了報告の末尾に必ず以下を記載すること:

  「**[Firecrawlスキャン依頼]** 以下のURLをChat側でスキャンしてください:」
  - 変更したHTMLファイルに対応する本番URL
  - そのページと関連するページ

  バックエンドのみの変更（DB/EF/API）ではこのセクションは不要。
```

## 旧テンプレートとの差分

| 項目 | 旧（v1） | 新（v2） |
|---|---|---|
| 指示の書き方 | ルールなし | S-01: 具体的なファイルパス・行番号必須。曖昧委譲禁止 |
| スコープ制限 | なし | S-05: 依頼外改善禁止、早すぎる抽象化禁止 |
| コメント | なし | S-06: WHYのみ。WHAT禁止 |
| 実行前チェック | 5項目 | 6項目（S-07: 参照先の実在確認を追加） |
| 完了報告 | 3行 | S-02/S-03/S-04/S-16を統合。検証方法の明記を義務化 |

---

## 適用対象

このテンプレートは以下の全てに適用する：
- CC依頼文（④ CC Brief Writerが作成するもの）
- CCへの追加指示・質疑回答
- CCの完了報告フォーマット（CC側に遵守させる）

## Chatセッションでの使い方

Chatでの④ CC Brief Writer は、CC依頼文を作成する際に必ずこのテンプレートに従う。
テンプレートに含まれないCC依頼固有の情報（テスト項目、デプロイ手順等）は自由に追加してよい。
