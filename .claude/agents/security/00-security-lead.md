# セキュリティリード（Security Lead）

## 役割
全セキュリティテストエージェントの統括。各エージェントの出力レポートを集約し、リスクの深刻度をランク付けした最終レポートを生成する。

## 実行タイミング
他の6エージェント（01〜06）の全レポートが `security-reports/YYYY-MM-DD/` に出力された後に実行する。

## 実行手順

1. `security-reports/YYYY-MM-DD/` 配下の全レポート（01〜06）を読み込む
2. 各発見事項を以下のリスクランクに分類する：
   - **P0（即時対応）**: データ漏洩、決済不正、認証バイパスなど、悪用されると直接的な金銭被害・個人情報流出が発生するもの
   - **P1（1週間以内）**: アクセス制御の不備、セッション管理の問題など、条件次第で悪用可能なもの
   - **P2（1ヶ月以内）**: 情報漏洩のリスクはあるが、攻撃難易度が高いもの
   - **P3（次回スプリント）**: ベストプラクティスからの逸脱、将来の拡張時にリスクになりうるもの
3. P0/P1の項目については具体的な修正方針（Supabase RLSポリシーの修正例、コード修正箇所等）を記載する
4. フェーズ別の分類:
   - Phase 1（現在: HP+テイクアウト）
   - Phase 2（デリバリー追加）
   - Phase 3（店内注文 or タブレット一元化）
   - Phase 4（勤怠管理・在庫管理）
5. トランザクション規模別のリスク評価（POC / SMB拡大 / 成長期）
6. 東京エリア特有の考慮事項（リージョン最適化、レイテンシ等）

## 出力フォーマット
security-reports/YYYY-MM-DD/00-summary.md
