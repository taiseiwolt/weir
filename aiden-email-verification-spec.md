# Weir メール認証フロー仕様書

文書番号: SPEC-EMAIL-VERIFY-001
作成日: 2026-03-21
ステータス: 確定（実装待ち）

## 1. 方針サマリ

| 項目 | 決定内容 |
|------|---------|
| 認証方法 | 登録後にメール送信 → URL クリックで認証完了 |
| 認証期限 | 60分以内 |
| 未認証ユーザー | ログイン不可（認証完了まで一切使えない） |
| 期限切れ | アカウント削除（再登録が必要） |
| 既存会員 | 全員に認証メールを送る |

---

## 2. ユーザーフロー

### 新規会員登録

```
会員登録画面
  → 必要情報入力（氏名、メール、電話番号、パスワード等）
  → 「登録する」ボタン
  → Supabase Auth でアカウント作成（email_confirmed_at = NULL）
  → 認証メール自動送信
  → 完了画面表示:
    「認証メールを送信しました」
    「60分以内にメール内のURLをクリックして登録を完了してください」
    「メールが届かない場合は迷惑フォルダをご確認ください」
    「認証メールを再送する」ボタン

--- ユーザーがメールを確認 ---

  → メール内のURLをクリック
  → Supabase Auth が email_confirmed_at を更新
  → 認証完了ページ表示:
    「会員登録が完了しました」
    「ログイン」ボタン
```

### ログイン（認証済み）

```
ログイン画面
  → メール + パスワード入力
  → Supabase Auth で認証
  → email_confirmed_at が NULL でないか確認
  → NULL でなければ → ログイン成功
```

### ログイン（未認証）

```
ログイン画面
  → メール + パスワード入力
  → Supabase Auth で認証
  → email_confirmed_at が NULL → ログイン拒否
  → エラー表示:
    「メール認証が完了していません」
    「認証メールを再送する」ボタン
```

---

## 3. 認証期限切れの処理

| タイミング | 処理 |
|-----------|------|
| 登録後60分経過 & 未認証 | pg_cronバッチでアカウント削除 |
| 削除対象の条件 | `email_confirmed_at IS NULL AND created_at < NOW() - INTERVAL '60 minutes'` |
| バッチ実行頻度 | 毎時（pg_cron） |
| 削除対象 | Supabase Auth のユーザー + members テーブルの該当レコード |

### 期限切れ後にログイン試行した場合

```
ログイン画面
  → メール + パスワード入力
  → アカウントが存在しない → 「アカウントが見つかりません」
  → ユーザーは新規登録からやり直し
```

## 4. 認証メール再送

| 項目 | 内容 |
|------|------|
| 再送可能回数 | 60分以内に最大3回 |
| 再送ボタン表示場所 | 登録完了画面 + ログイン拒否画面 |
| 再送時の期限 | 最初の登録時刻から60分（再送しても期限は延長しない） |

---

## 5. 既存会員への対応

| 手順 | 内容 |
|------|------|
| 1 | 全既存会員に認証メールを一括送信 |
| 2 | メール内容: 「セキュリティ強化のためメール認証をお願いします」 |
| 3 | 認証期限: 既存会員は7日間の猶予を設ける（新規の60分とは別扱い） |
| 4 | 7日後も未認証: ログイン不可（認証メール再送で対応可能、アカウント削除はしない） |

※ POC前でテストアカウント程度のため、実質的な影響は軽微

## 6. 認証メールの内容

### 件名
```
【Weir】メールアドレスの認証をお願いします
```

### 本文（概要）
```
○○様

Weirへのご登録ありがとうございます。
以下のURLをクリックして、メールアドレスの認証を完了してください。

[認証URL]

※ このURLの有効期限は60分です。
※ 期限を過ぎた場合は再度会員登録をお願いいたします。

心当たりがない場合は、このメールを無視してください。
```
