## 1. 方針サマリ

| 項目 | 決定内容 |
|------|---------|
| 退会導線 | MyPageに「退会」ボタン |
| ポイント残高 | 即時失効 |
| 進行中の注文 | 退会予約として受付 → 全注文完了後に自動退会処理 |
| 再登録 | 同じメールアドレスで再登録OK |
| クーリングオフ | なし（退会ボタン → 確認画面 → 完了） |

## 2. ユーザーフロー

### 通常パターン（進行中注文なし）

```
MyPage → 「退会する」ボタン
  → 確認画面表示
    - 「ポイント残高（○○pt）は即時失効します」
    - 「退会後90日間はデータを保持し、その後匿名化されます」
    - 「注文履歴は法令上7年間保持されます（個人情報との紐付けは解除されます）」
  → 「退会する」ボタン（赤色、2回目の確認）
  → 退会完了画面
    - 「退会が完了しました」
    - ログアウト処理（セッション破棄）
    - トップページへのリンク表示
```

### 退会予約パターン（進行中注文あり）

```
MyPage → 「退会する」ボタン
  → 確認画面表示
    - 「現在進行中の注文が○件あります」
    - 「全ての注文が完了次第、自動的に退会処理を行います」
    - 「ポイント残高（○○pt）は退会処理時に失効します」
  → 「退会予約する」ボタン
  → 退会予約完了画面
    - 「退会予約を受け付けました」
    - 「進行中の注文が完了次第、退会処理が実行されます」
    - 退会予約中も通常通りサービス利用可能（新規注文は不可）
  
  --- 全注文完了後 ---
  → 自動退会処理実行
  → 退会完了メール送信
```

## 3. データ処理（SEC-001 データ保持ポリシーに準拠）

### 退会実行時に即時処理

| データ | 処理 |
|--------|------|
| ポイント残高 | 全額失効（point_transactionsにexpired記録） |
| 未使用クーポン | 無効化 |
| 配達先住所 | 物理削除（DELETE） |
| セッション | 全セッション無効化 |
| カート内容 | 削除 |

### 退会後90日間保持 → 匿名化

| データ | 90日後の処理 |
|--------|------------|
| 氏名 | ハッシュ値に置換 |
| メールアドレス | ハッシュ値に置換 |
| 電話番号 | ハッシュ値に置換 |

### 長期保持（税法上の義務）

| データ | 保持期間 |
|--------|---------|
| 注文履歴 | 注文日から7年間（匿名化後は個人紐付け解除） |
| 決済記録 | 同上 |

## 4. DB設計（案）

### membersテーブルへの追加カラム

| カラム名 | 型 | 説明 |
|---------|---|------|
| withdrawal_status | TEXT | NULL（通常）/ 'pending'（退会予約中）/ 'withdrawn'（退会済み） |
| withdrawal_requested_at | TIMESTAMPTZ | 退会申請日時 |
| withdrawal_completed_at | TIMESTAMPTZ | 退会処理完了日時 |
| anonymized_at | TIMESTAMPTZ | 匿名化完了日時 |

### 退会予約→自動処理の仕組み

- pg_cronで毎時チェック: `withdrawal_status = 'pending'` のユーザーで、進行中注文が0件になったら退会処理実行
- または: 注文ステータスが「完了」に変わるトリガーで、該当ユーザーの退会予約をチェック

### 匿名化バッチ（月次）

- pg_cronで月次実行: `withdrawal_completed_at` から90日経過したレコードの PII をハッシュ化

---

## 5. 再登録の扱い

- 時期を問わず、常に**完全新規アカウント**として扱う
- ポイント・注文履歴・クーポン等の引き継ぎは一切なし
- 退会後90日以内の場合: 旧アカウントのPIIがまだDB上に残っているため、Supabase Auth上で旧アカウントを削除してから新規作成する（同一メールの重複を防ぐため）
- 退会後90日以降の場合: メールはハッシュ化済みのため、通常の新規登録フローでOK

## 6. 退会予約中の制限

| 操作 | 可否 |
|------|------|
| 進行中注文のトラッキング閲覧 | ○ |
| 進行中注文のキャンセル | ○ |
| 新規注文 | ✕（ブロック） |
| プロフィール編集 | ✕ |
| ポイント利用 | ✕ |
| 退会予約の取り消し | ○（「退会予約を取り消す」ボタン表示） |

## 7. 通知

| タイミング | 通知方法 | 内容 |
|-----------|---------|------|
| 退会完了時 | メール | 退会完了の確認 + 90日間データ保持の説明 |
| 退会予約時 | 画面表示 | 予約受付の確認（メールは不要、画面で完結） |
| 匿名化完了時 | なし | メールアドレスが既にハッシュ化されているため通知不可 |
